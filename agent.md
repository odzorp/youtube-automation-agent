# YouTube Automation Agent — Agents Overview

Short summary
- A local-first YouTube automation system that uses specialized AI "agents" to research topics, write scripts, design thumbnails, optimize SEO, manage production/publishing, and analyze performance.

Core agents
- `agents/content-strategy-agent.js`: discovers trending topics, builds content strategy and calendar.
- `agents/script-writer-agent.js`: generates video scripts and titles optimized for watch-time.
- `agents/thumbnail-designer-agent.js`: creates thumbnails and supports A/B testing.
- `agents/seo-optimizer-agent.js`: finds keywords and optimizes titles/descriptions/tags.
- `agents/publishing-scheduling-agent.js`: schedules uploads and manages publish flows.
- `agents/production-management-agent.js`: handles processing, encoding, and packaging content.
- `agents/analytics-optimization-agent.js`: collects analytics and recommends improvements.

Key files
- `index.js`: main application that initializes DB, credentials, agents, API endpoints and scheduler.
- `oauth-server.js`: helper to run OAuth flow and persist YouTube tokens to `config/tokens.json`.
- `setup.js` / `utils/credential-manager.js`: interactive credential setup helpers.
- `config/credentials.json`: (user-provided) YouTube & AI provider credentials. See `config/credentials.example.json`.
- `package.json`: available npm scripts (start, setup, scheduler, agent:* helpers).

Quick setup
1. Install dependencies:

```
npm install
```

2. Copy example credentials and configure:

```
cp config/credentials.example.json config/credentials.json
npm run credentials:setup
```

3. Complete OAuth (opens URL printed by `oauth-server.js`):

```
node oauth-server.js
```

4. Start the system:

```
npm start
```

Useful npm scripts
- `npm start` — start main application (`index.js`).
- `npm run setup` — run interactive setup.
- `npm run scheduler` — run scheduled automation (`schedules/daily-automation.js`).
- `npm run test` — run quick test harness (`test.js`).
- `npm run agent:strategy` / `agent:script` / `agent:thumbnail` / `agent:seo` / `agent:production` / `agent:publishing` / `agent:analytics` — run individual agents for debugging.

Storage & logs
- Generated assets and uploads: `data/` and `uploads/`.
- Local DB: `database/` (see `database/db.js`, sqlite3). 
- Logs: `logs/` (winston-based logger via `utils/logger.js`).

Notes & recommendations
- Ensure `NODE_ENV` and `.env` variables are set (see `.env` usage in code).
- Use `OPENAI_API_KEY` or `GEMINI_API_KEY` in environment for AI provider.
- The OAuth server writes tokens to `config/tokens.json` after authorization.
- Keep API credentials private; do not commit `config/credentials.json` or `config/tokens.json`.

Token encryption (new)
- `utils/token-crypto.js` provides AES-256-GCM encryption for `config/tokens.json`.
- Set `TOKEN_ENCRYPTION_KEY` (or fall back to `JWT_SECRET`) in `.env`.
- On first load, plaintext tokens are automatically migrated to encrypted form.
- YouTube tokens are auto-persisted on refresh via the `tokens` event listener in `utils/credential-manager.js`.
- Run `node test-token-crypto.js` to verify the encrypt/decrypt round-trip.
